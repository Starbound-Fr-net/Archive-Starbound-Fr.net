]]> Salut les gens,
Voici la traduction du post du 17 février, rédigé par Metadept.

Comme beaucoup d’entre vous s’en sont déjà aperçu, les modifications apportées par la mise à jour instable du 12 Février ont été appliquées à la version stable. Nous avons changé la façon qu’a le serveur de configurer les comptes utilisateurs, et étant donné que cela nous a beaucoup été demandé, voici une brève explication.

Une entrée serverUsers vide devrait désormais figurer dans le fichier starbound.config de votre serveur. C’est ici que vous pourrez ajouter des comptes de joueurs et d’administrateurs. Voici un exemple de configuration : 

  "serverUsers" : {
    "fred" : {
      "admin" : true,
      "password" : "hunter2"
    },
    "george" : {
      "admin" : false,
      "password" : "swordfish"
    }
  },
  "allowAnonymousConnections" : false,
  "allowAdminCommands" : true,
  "allowAdminCommandsFromAnyone" : false,
  "bannedIPs" : [  ],
  "bannedUuids" : [  ],

Notez que les comptes d’utilisateurs des admins et de non-admins sont configurés au même endroit, et que tous ces comptes nécessitent chacun un nom d’utilisateur (à entrer en jeu lorsque vous vous connectez au serveur). Plusieurs utilisateurs peuvent utiliser le même compte en même temps, du coup si vous voulez simplement mettre un mot de passe à votre serveur, vous pouvez créer un compte nommé par exemple ‘guest’, avec un mot de passe unique pour tous les non-admins.

De manière à forcer tous les utilisateurs à s’identifier, vous devrez régler allowAnonymousConnections sur false dans le fichier starbound.config de votre serveur. Si allowAnonymousConnections est sur true (Il s’agit du réglage par défaut), n’importe quel joueur pourra se connecter au serveur sans fournir de nom d’utilisateur ou de mot de passe. Les administrateurs, bien entendu, auront toujours besoin de s’identifier pour bénéficier de leurs privilèges si ceux-ci sont restreints via le réglage allowAdminCommandsFromAnyone.

Nous avons aussi réglé les problèmes avec la liste noire, de manière à ce que les utilisateurs puissent être bannis par UUID ou par IP. Si vous voulez dé-bannir un joueur qui a été banni de manière permanente, vous devrez effacer son nom de la liste noire manuellement et relancer le serveur (pour l’instant).

J’espère que cela vous aura permis de mieux comprendre ce nouveau système !

Pour accompagner le serveur de jeu, un serveur vocal à été créé.
D’ailleurs le débat pour choisir entre Mumble et Teamspeak fut long et acharné :

– Purexo : Mumble c’est nul, et une personne sur deux à des bugs avec. Les gens savent pas régler le son avec, et le son est dégueu.
– Purexo : Teamspeak, c’est joli et bien foutu. il y a des addons, des thèmes etc… Et ça bug pas pour un rien.
– Oniricorpe : J’m’en tape lel
– Purexo : OK, TS donc ^^.

Ainsi pour ce connecter, rien de plus simple :

  • Teamspeak : ts3server://starbound-fr.net?password=human ou ici
    • Adresse : starbound-fr.net
    • Port : 9987 (par défaut)
    • Mot de passe : human
  • Starbound
    • Adresse : starbound-fr.net
    • Pas de compte ni de mot de passe 😉

Il est obligatoire de se connecter avec un personnage dédié à ce serveur, donc vierge au début.

Sachez qu’il est pour le moment en libre accès, mais que si nous voyons qu’il y a des dérapages, nous passerons au système de candidatures, comme prévu initialement.
