[Guide sur le modding]
[Windows]

   
Citation :Bienvenue dans mon guide sur le modding celui si n'est pas forcement complet mais je m’efforcerais de le rendre un maximum exploitable.
Avant de commencer je precise que Starbound est très flexible donc les modification sont extrêmement simple et n'importe qui peut se lancer.
Bien sur pour crée un mod qui va revolutionner le jeu il faudras mettre les main dans le LUA et pour ça je ne vous suit d'aucune utilité x).
   
Sommaire :
  • Recherche du dossier racine
  • Le Dossier Assets
   
[Recherche du dossier racine]
Le dossier racine c'est le dossier ou est installer le jeu c'est la bas que toute les modifications vont être effectué en générale il se trouve ici :
Code :
"C:\Program Files (x86)\Steam\SteamApps\common\Starbound"
   


Vous devriez avoir à peu près les mêmes dossiers que moi :
   

[Le Dossier Assets]
Le dossier Assets est le dossier qui va nous intéresser le plus c'est la bas qu'il y a tout les fichiers du jeu
Alerte Spoiler de Noob:

Comme vous pouvez le constater si vous fouillez un peu le contenu est riche mais dans le cadre du tuto nous allons simplement nous intéresser à 2 ou trois dossiers et fichiers dans tout ce fouillis organisé.
   

[La création d'une nouvelle pioche]
Nous allons donc pour expliquer les rudiments du modage sur starbound créer une nouvelle pioche
Pour cela nous allons aller dans :
Code :
"C:\Program Files (x86)\Steam\SteamApps\common\Starbound\assets\items\tools"
   
Comme vous pouvez le constater il y a tout les outils présent dans starbound:
Alerte Spoiler de Noob:

   
Ici pour nous éviter les premières lignes de code nous allons récupérer par exemple l'item de la pioche en diamant ainsi que ces textures:
Alerte Spoiler de Noob:

   
En ce qui concerne les texture je vous laisse faire garder juste le nom des images nous les modifieront plus tard.
Ensuite ouvrer le fichier "diamondpickaxe.miningtool" avec notepad++ ou votre bloc note favoris :p
Cela devrait vous donner ça ! :
Code :
{

Alors là on touche au gros du truc :p
Comme vous pouvez le voir cela ne ressemble presque pas à du code et si vous avez quelques connaissances en anglais vous comprendrez assez bien chaque ligne.
   
Mais pour les autres voilà une mini traduction:
Alerte Spoiler de Noob:

     "itemName" : Nom de l'objet
     "inventoryIcon" : Icone de l'image
     "dropCollision" : Aucune idée a vous de faire des tests
     "maxStack" : Combien d'objet empilable
     "rarity" : Rareter de l'objet
     "inspectionKind" : Groupe de l'objet
     "description" : Description de l'objet
     "shortdescription" : Courte description de l'objet
     "largeImage" : Grande image de l'objet
     "image" : Image de l'objet
     "handPosition" : Position de l'objet dans la main du joueur
     "fireTime" : Vitesse de l'objet
     "swingStart" : Angle de l'image au demarrage de l'animation
     "swingFinish" : Angle de l'image à la fin de l'animation
     "blockRadius" : Le rayon d'action de l'objet
     "altBlockRadius" : Le rayon d'action de l'objet quand on appuie sur "ALT"
     "twoHanded" : Si c'est un objet a une ou 2 main
     "strikeSound" : Bruit ou son que produit l'objet quand celui si est utiliser
     "durability" : Durabilité de l'objet
     "durabilityPerUse" : Le nombre de point de durabilité enlever quand utiliser
     "durabilityRegenChart" : Combien redonne chaque minerais pour la réparation de l'item
     "tileDamage" : Le taux de dégâts causés au bloc
     "tileDamageBlunted" : Le taux de dégâts causés au bloc quand la pioche est usée
   
Voilà maintenant passons à la modification pure et dur de l'objet :p
Pour cela il n'y a rien de bien compliquer il vous suffit de modifier les dites valeur précédemment traduite :p
   
Pour la pioche que j'ai créée sur un autre topic voilà les valeurs :
Code :
{
     "itemName" : "powereddiamondpickaxe",
     "inventoryIcon" : "powereddiamondpickaxeicon.png",
     "dropCollision" : [-4.0, -3.0, 4.0, 3.0],
     "maxStack" : 1,
     "rarity" : "common",
     "inspectionKind" : "tool",
     "description" : "The god Pickaxe !",
     "shortdescription" : "Powered Diamond Pickaxe",
     "largeImage" : "powereddiamondpickaxe.png",
     "image" : "powereddiamondpickaxe.png",
     "handPosition" : [-3, -4],
     "fireTime" : 0.28,
     "swingStart" : -40,
     "swingFinish" : 60,
     "blockRadius" : 4,
     "altBlockRadius" : 1,
     "twoHanded" : true,
     "strikeSound" : "/sfx/tools/pickaxe_hit.wav",
     "durability" : 7500,
     "durabilityPerUse" : 1,
     "durabilityRegenChart" : [
       [ [ "copperore" ], 750], [ [ "silverore" ], 1125], [ [ "goldore" ], 1500], [ [ "platinumore" ], 1875], [ [ "diamond" ], 2250]
     ],
     "tileDamage" : 2.7,
     "tileDamageBlunted" : 0.6
}
   
Et c'est là que je vous demande de faire très attention car si il manque ne serais-ce qu'une virgule votre item ne serais même pas détecté !
   
Comme dit plus haut je vous est demander de garder les nom des texture nous allons maintenant les modifier :p
C'est là qu'il va falloir être rigoureux car il faut que le nom des images corresponde avec ce que vous avez marqué dans le fichier de la pioche
   
Exemple :
   "itemName" : "powereddiamondpickaxe",
   "image" : "powereddiamondpickaxe.png"
   "largeImage" : "powereddiamondpickaxe.png"
   "inventoryIcon" : "powereddiamondpickaxeicon.png"
   
   
Vous voyez bien que chaque ligne correspond bien a chaque image ainsi que le nom de l'item au nom du fichier
